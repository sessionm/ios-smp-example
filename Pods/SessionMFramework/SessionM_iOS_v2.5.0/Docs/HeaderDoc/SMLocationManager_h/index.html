Classes

SMLocationManager

SessionM Location service interface. This is the main class in SessionM Location iOS SDK. It defines methods for all location based operations such as location and region monitoring.

extern NSString * const SMLocationManagerDidTriggerLocation NS_SWIFT_NAME(
    triggeredLocationEventNotification);
Discussion

Information about the location event and the user's location when the event was triggered can be accessed from the notification's userInfo property.

NS_SWIFT_NAME(disabledLocationServicesNotification)

This notification is sent when location services are disabled. 

extern NSString *const SMLocationManagerLocationServicesDisabled NS_SWIFT_NAME(
    disabledLocationServicesNotification);
Discussion

Whenever the location service has been started but the device does not have permission to use location services, this notification is fired.

NS_SWIFT_NAME(disabledWhenInUseLocationServicesNotification)

This notification is sent when region monitoring fails due to not having "when in use" permission. 

extern NSString *const SMLocationManagerWhenInUseLocationServicesDisabled NS_SWIFT_NAME(
    disabledWhenInUseLocationServicesNotification);
Discussion

When this notification is recieved, region monitoring did not start because "when in use" permission was not granted. Simply call startGeofenceService to try again. The notification's userInfo will have the @"error" key set to the error.

NS_SWIFT_NAME(updatedLocationEventsNotification)

Notifies observers that the currently monitored location events were updated. 

extern NSString * const SMLocationManagerDidUpdateGeoLocations NS_SWIFT_NAME(
    updatedLocationEventsNotification);
Discussion

Currently monitored location events can be accessed from the notification's userInfo property with the kSMLocationManagerLocationsKey key. Currently monitored geofence regions can be accessed with the kSMLocationManagerRegionsKey key.
