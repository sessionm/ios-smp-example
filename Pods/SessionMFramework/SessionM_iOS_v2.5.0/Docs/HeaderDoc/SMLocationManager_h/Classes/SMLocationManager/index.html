Introduction

SessionM Location service interface. This is the main class in SessionM Location iOS SDK. It defines methods for all location based operations such as location and region monitoring.

refreshGeofenceService

Refreshes region monitoring based on SessionM Platform portal configurations. 

- (void)refreshGeofenceService;
Discussion

This call made on the [SMLocationManager sharedInstance] object will refresh monitoring regions based on configurations setup by the developer in the SessionM Platform portal. Regions monitored persist across app launches and are only cleared via stopGeofenceService.

startGeofenceService

Starts region monitoring. 

- (void)startGeofenceService;
Discussion

This call made on the [SMLocationManager sharedInstance] object will start monitoring regions.

startLocationService

Starts location updates. 

- (void)startLocationService;
Discussion

This call made on the [SMLocationManager sharedInstance] object will start location updates. If permission is denied a SMLocationManagerLocationServicesDisabled notification will be fired.

stopGeofenceService

Stops all region monitoring. 

- (void)stopGeofenceService;
Discussion

This call made on the [SMLocationManager sharedInstance] object will stop monitoring all regions.

stopLocationService

Stops location updates. 

- (void)stopLocationService;
Discussion

This call made on the [SMLocationManager sharedInstance] object will stop location updates. This does not clear monitored regions that persist across app launches.
