Introduction

This API manager contains methods for registering for and handling remote notifications.

handleRemoteNotification:handler:payload:handler:

Checks if the given remote notification was sent by the SessionM Platform. 

- (BOOL)handleRemoteNotification:(NSDictionary<NSString *, NSObject *> *)payload 
        handler:(_Nullable NotificationHandler)handler NS_SWIFT_NAME(handleRemoteNotification(
                payload:handler:));
Parameters
payload

The NSDictionary payload data.

handler

The handler block to be called after processing the notification payload.

Return Value

BOOL indicating whether notification was sent by the SessionM Platform.

Discussion

This method should be called from the developer's application:didReceiveRemoteNotification: and application:didFinishLaunchingWithOptions: delegate methods with the respective userInfo or launchOptions payload. If the payload contains a SessionM Platform notification, then an SMNotificationMessage object will be returned to the developer via the handler parameter - and the success boolean will be set to YES.

storeDeviceToken:_:

Stores user's Apple Push Notification Service device token (required for users to receive remote notifications through APNS). 

- (void)storeDeviceToken:(NSData *)token NS_SWIFT_NAME(storeDeviceToken(
                _:));
Parameters
token

The user's device token.

Discussion

This method should be called with the given device token from the developer's application:didRegisterForRemoteNotificationsWithDeviceToken: delegate method. Note: this method cannot be used simultaneously with storePushRegistrationToken:.

storePushRegistrationToken:_:

Stores user's Firebase or Google Cloud Messaging registration token (required for users to receive remote notifications through FCM/GCM). 

- (void)storePushRegistrationToken:(NSString *)token NS_SWIFT_NAME(storePushRegistrationToken(
                _:));
Parameters
token

The user's registration token.

Discussion

This method should be called with the given registration token from the developer's messaging:didRefreshRegistrationToken: delegate method (or if using Google Cloud Messaging - from the registration handler in any calls to tokenWithAuthorizedEntity:scope:options:handler:). Note: this method cannot be used simultaneously with storeDeviceToken:.

Properties

hasPendingNotificationAction

The most recent notification passed to handleRemoteNotification:handler:.

hasPendingNotificationAction

@property(nonatomic,
    assign,
    readonly) BOOL hasPendingNotificationAction;
Discussion

If a notification with an action was received but has not yet been executed, then this property will return YES. Otherwise it will return NO.

pendingNotification

The most recent notification passed to handleRemoteNotification:handler:. 

@property(nullable,
    nonatomic,
    strong,
    readonly) SMNotificationMessage *pendingNotification;
Discussion

This property will be set to nil after a successful call to executePendingNotificationAction or executeActionForNotification: .
